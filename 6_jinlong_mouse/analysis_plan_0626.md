## 5. Scripts Description

### Script 1 — `1_produce_nf-core_Samplesheet.py`
Generates `nf_core_samplesheet.csv` (columns: `sample`, `fastq_1`, `fastq_2`, `strandedness`).  
- Reads `../jinlong.csv`
- Maps numeric name → `J_{name}` directory; letter names kept as-is
- Verified: 12/12 samples found, all R1/R2 pairs located

**Run:** `cd scripts && python 1_produce_nf-core_Samplesheet.py`

---

### Script 2 — `2_nextflow.sh`
Launches nf-core/rnaseq 3.15.1 in a detached tmux session named `rnaseq`.

Key parameters:
```
--aligner star_salmon          # STAR alignment + Salmon quantification
--gencode                      # GENCODE annotation format
--max_cpus 28 / --max_memory 108.GB
```
Log: `scripts/nextflow_run.log`

**Run:** `bash 2_nextflow.sh`  
**Monitor:** `tmux attach -t rnaseq` or `tail -f scripts/nextflow_run.log`

### Script 3 — `3_Optional_work_monitor.sh`
Convenience script to check tmux session, log tail, and output file status.

### Script 4 — `4_run_DE_PCA.R`
Main DE analysis. Run with `conda activate DE_R45` then `Rscript 4_run_DE_PCA.R`.

**Pipeline:**
1. Load `jinlong.csv`, build meta table with `sample_id` and `Group` (G4 = reference factor level)
2. Read `salmon.merged.gene_counts.tsv`
3. Gene filtering (two-tier):
   - **Ribosomal regex:** `^Rpl|^Rps|^Mrpl|^Mrps` (mouse naming, case-insensitive)
   - **Non-coding regex:** `^mt-`, snoRNA, lncRNA, miRNA, Gm-prefixed predicted genes
   - **Low-count filter:** ≥10 counts in at least (n−2) samples
4. DESeq2 model: `~Group`, G4 as intercept
5. `lfcShrink(..., type="ashr")` for all contrasts
6. Significance threshold: `padj ≤ 0.05 & |log2FC| ≥ 0.263` (20% FC)
7. Outputs: PCA, volcano plots, top-50 heatmaps, DEG CSVs
8. **Saves `.rds` objects** (`res_list`, `vsd`, `meta`, `counts_raw`) for script 5

**Note:** No mouse gene annotation xlsx available → regex fallback only. If a mouse annotation file is obtained later, add it analogous to the human version in the ZhenYan project.

### Script 5 — `5_run_enrichment.R`
Enrichment analysis. Depends on `.rds` files from script 4.

#### 5a. GO Enrichment (`enrichGO`)
- Ontologies: BP, MF, CC
- Gene sets: UP, DOWN, ALL significant DEGs
- Background: all expressed genes (after low-count filter)
- Outputs: CSV + dotplot PDF per ontology × direction × comparison

#### 5b. KEGG Pathway (`enrichKEGG`)
- Organism: `mmu` (mouse KEGG)
- Gene sets: UP, DOWN, ALL
- Gene names added via `setReadable()`

#### 5c. GSEA
Three GSEA analyses per comparison:

| Analysis | Gene set source | Method |
| :--- | :---: | :---: |
| GSEA GO BP | org.Mm.eg.db | `gseGO()` |
| GSEA KEGG | KEGG mmu | `gseKEGG()` |
| GSEA Hallmark | MSigDB H collection | `GSEA()` + msigdbr |

**Rank statistic:** `sign(log2FC) × (−log10(pvalue))`  
Outputs: CSV, ridgeplot (GO), dotplot split by sign (KEGG/Hallmark), individual GSEA curve PDFs for top 3 enriched + top 3 depleted terms.

#### 5d. Stem Cell Marker Analysis

**Curated panel (6 categories):**

## 7. Key Parameters & Rationale

| Parameter | Value | Rationale |
| :--- | :---: | :---: |
| Strandedness | reverse | Standard Illumina TruSeq PE library |
| Aligner | star_salmon | Best practice for quantification accuracy |
| `--gencode` flag | yes | GENCODE M35 annotation uses GENCODE format |
| DESeq2 reference level | G4 | Control group per client instruction |
| LFC shrinkage | ashr | More accurate for small n, handles outliers |
| Significance: padj | ≤ 0.05 | Standard FDR threshold |
| Significance: \|log2FC\| | ≥ 0.263 | 20% fold change minimum |
| Low-count filter | ≥10 counts in (n−2) samples | Removes noise while tolerating 2 outlier samples |
| GSEA min/max GS size | 15 / 500 | Standard range; avoids very small/generic sets |
| KEGG organism | mmu | Mus musculus KEGG code |

## 9. Notes & Known Limitations

- **No mouse gene annotation xlsx:** The reference project uses `human_Gene_annotation_20260202.xlsx` for biotype-based filtering. No equivalent mouse file exists. Script 4 falls back to regex filtering (ribosomal + non-coding patterns). If a mouse annotation file is created later, add it as `PRIMARY_METHOD` following the same pattern as the ZhenYan script.
- **Group labels G1–G4 are generic:** The actual biological meaning of groups 1–3 is not specified in `jinlong.csv`. Rename in the contrasts list once the client clarifies.
- **Stem cell type unknown:** Script 5 uses a broad 6-category marker panel. After discussing biology with the client, the panel can be narrowed to the relevant stem cell type.
- **GSEA requires ≥100 ranked genes:** If a comparison has very few expressed genes, GSEA will be skipped with a warning.
- **MSigDB C8 ORA** depends on msigdbr fetching data from remote; ensure internet access when running script 5 for the first time (data is cached after first download).
