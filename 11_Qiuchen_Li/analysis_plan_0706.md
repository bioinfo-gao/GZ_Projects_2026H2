## 6. Key Parameters & Rationale

- **Comparison design:** All 3 pairwise contrasts (Mix vs NT, A5BKO vs NT, A5BKO vs Mix) — confirmed with user; no single reference group.
- **Gene filter:** GENCODE `gene_type == protein_coding` via `human_Gene_annotation` xlsx (falls back to regex if xlsx unavailable).
- **Low-expression filter:** ≥10 counts in ≥(n−2) samples.
- **Significance threshold:** padj ≤ 0.05 & |log2FC| ≥ 0.263 (log2(1.2), ≥1.2-fold).
- **LFC shrinkage:** ashr.
- **No mouse-specific STAR twopass override** — human aligns with nf-core defaults.

## 9. REDO (2026-07-06): sample exclusion + heatmap annotation bug fix

**Trigger:** client reviewed the PCA plot and flagged that sample A5BKO_2 appeared to cluster with
the NT group rather than with its own A5BKO replicates — a likely mislabeling, not normal
biological/technical noise. Client also flagged that all 3 delivered heatmaps had their Group
legend and sample names swapped/reversed at the bottom axis.

**Issue 1 — A5BKO_2 mislabeling, confirmed:**

| Sample | PC1 | PC2 |
| :--- | :---: | :---: |
| NT_1 | 5.01 | -3.61 |
| NT_2 | 4.94 | -3.58 |
| NT_3 | 4.94 | -3.40 |
| **A5BKO_2** | **4.94** | **-3.72** |
| A5BKO_1 | 10.63 | 5.48 |
| A5BKO_3 | 10.73 | 5.47 |

A5BKO_2's coordinates are essentially indistinguishable from the 3 NT replicates and far from its
own A5BKO_1/A5BKO_3 (which cluster tightly together). Excluded from this redo; A5BKO group is now
n=2 (A5BKO_1, A5BKO_3).

**Issue 2 — heatmap Group/sample-name reversal, confirmed and root-caused:**

The heatmap annotation code built `ann_df` via logical-subset filtering:
```r
ann_df <- data.frame(Group = as.character(meta$Group[meta$sample_id %in% smp_comp]),
                     row.names = smp_comp)
```
This preserves **meta's own row order** for the extracted Group values, not `smp_comp`'s order
(treatment-then-control). Whenever meta's original sheet order didn't happen to already place the
treatment group before the control group, the two got out of sync. Verified by reconstructing the
exact objects from `rds_cache/meta.rds`:
- **Mix_vs_NT**: correct by coincidence (Mix appears before NT in both the original sheet and in
  `smp_comp`'s trt-then-ctrl order).
- **A5BKO_vs_NT**: **reversed** — A5BKO_1/2/3 were labeled "NT" and NT_1/2/3 were labeled "A5BKO" in
  the delivered heatmap.
- **A5BKO_vs_Mix**: **reversed** — A5BKO_1/2/3 were labeled "Mix" and Mix_1/2/3 were labeled "A5BKO".

**Fix applied** (in `4_run_DE_PCA.R`, heatmap section): replaced the logical-subset filter with
`match(smp_comp, meta$sample_id)`, which guarantees the Group value assigned to each row name is
looked up by that exact sample_id regardless of meta's internal row order. Additionally added a
hard `stopifnot()` assertion immediately after building `ann_df` that recomputes the expected Group
per sample_id and errors out if there is ANY mismatch — so a future regression of this exact bug (or
any similar one) fails loudly during the script run instead of silently producing a wrong heatmap
that could ship to a client. This pattern (verify-then-fail-loud, not just "be careful") has been
written into `/home/gao/.claude/commands/rnaseq.md` (checklist item 15) so it propagates to all
future RNAseq projects, not just this one.

**Result after fix:** DEG counts changed substantially now that the mislabeled sample is gone —
A5BKO_vs_NT went from 18 DEGs (previous, contaminated by the mislabeled A5BKO_2 diluting the true
A5BKO signal toward NT) to 3,413 DEGs. Mix_vs_NT and A5BKO_vs_Mix also increased modestly (5,478→5,911
and 6,212→6,937) from the cleaner 2-vs-3/3-vs-2 DESeq2 dispersion fit. PCA separation ratio (new
metric, see rnaseq.md checklist item 14) is now 182.57x — extremely clean group separation once the
mislabeled sample is removed, versus visibly overlapping before.
