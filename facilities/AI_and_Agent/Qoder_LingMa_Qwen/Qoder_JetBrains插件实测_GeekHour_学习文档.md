## 0. 一句话总结

Qoder 是**目前 JetBrains 生态里少见的"能打"的 AI 编程插件**:原生插件、国内免梯子、响应稳定,并且带**记忆(Memory)、规则(Rules)、指令、@database 上下文、MCP** 等进阶能力。视频用一个"从零生成个人博客后端工程"的完整实测,验证了它端到端的自主开发能力。

## 1. 背景:为什么是 Qoder

- 各类 AI 编程工具(Cursor、VSCode 系)体验都不错,但在 **JetBrains 全家桶里总差点意思**。
- 典型痛点:在 Cursor/VSCode 里生成代码后,还得切回 IDEA 调试运行、改细节,**来回切换很割裂**。
- Qoder 官方定位:**面向真实软件的智能体(Agentic)编程平台**。本视频聚焦它的 **JetBrains 插件**形态。

## 2. 安装与初始配置

### 2.1 安装插件

| 步骤 | 操作 |
| :--- | :--- |
| 1 | 在 JetBrains IDE(以 IDEA 为例)设置界面打开**插件市场** |
| 2 | 搜索 **Qoder** → 点击安装 |
| 3 | 安装完成后**重启 IDE** 生效 |
| 4 | 重启后 IDEA 右侧出现 **Qoder 侧边栏图标**,点击弹出交互界面 |

### 2.2 界面语言

右上角下拉菜单 → 插件设置最下方 → 把显示语言改为**中文**。

### 2.3 插件设置(常用功能)

在插件设置里可调:**自动补全、会话、网络代理** 等,按需调整。

### 2.4 个人设置(进阶能力入口)

"个人设置"里可配置 **MCP 服务、记忆(Memory)、规则(Rules)、指令**。

- **MCP 服务**:为智能体扩展工具/能力。点击进入 **MCP 广场** 选装各种 MCP 工具。
  - 示例:安装 **GitHub MCP** → 点安装 → 输入 Token 授权 → 智能体即可查询仓库、提交 PR 等。

### 2.5 其他使用形态

除 IDE 插件外,Qoder 还支持:**独立 App、命令行 CLI 工具、QoderWork** 等,覆盖不同场景。

## 3. 亮点能力

### 3.1 记忆(Memory)—— 自动理解项目

- 打开项目后**无需任何操作**,Qoder 就自动生成了 4 条记忆:
  1. 项目的 **Java 后端技术栈**
  2. **Maven 构建**信息
  3. **运行环境与安全配置**
  4. **后端核心功能与架构**
- 例:自动识别出后端基于 **Spring Boot + MyBatis Plus + MySQL + Redis**,并提取核心依赖、模块与架构设计。
- 后续交互中会**持续更新记忆库**,形成对个人编码习惯、项目特点、常见问题的记忆,让生成越来越精准。

### 3.2 规则(Rules)与指令

- **规则**:约束智能体行为——本质是一个规定"代码规范/结构/风格/功能要求"的 **Markdown 文件**。
  - 配置路径:个人设置 → 规则 → 新建规则 → 取名 → "立即添加"会打开一个 markdown 文件 → 把事先准备好的规则内容导入 → 保存。
- **指令**:定义快捷操作。

### 3.3 @database 上下文

在 IDEA 右侧数据源里配好数据库连接后,回到 Qoder 对话框点 **➕ → @database** 引用该库;Qoder 会**按真实表结构**生成更精准的实体类和后端代码。

## 5. 实测二:从零创建个人博客后端(端到端)

**整体分三步:**

1. **前端设计**:用 Qoder App 设计博客项目的前端功能与页面结构(用户注册/登录、文章发布/浏览、评论互动等)。
2. **数据库设计**:根据前端设计生成后端**数据库表结构与建表语句**。
3. **后端代码**:在 IDEA 里用 Qoder 插件、依据库表结构生成后端代码。

### 5.1 前端设计(Qoder App)

- Qoder App → 打开文件夹 → 新建空文件夹 `blog` 作为项目根目录 → 进入交互界面。
- 输入提示词让它设计前端功能与页面结构。
- **模式选择**:试用了**最高质量模式**(使用第一梯队的顶级模型,效果好但 **credits 消耗更多**)。
  - 省钱技巧:可在下方指定专门生成代码的 **coding plan**,与高质量模式**结合使用**——兼顾质量与成本。
- 结果:整体风格类似 GitHub,常见模块齐全。

### 5.2 数据库表结构

- 让它生成后端数据库表结构与建表语句 → 生成了 3 个 SQL 文件:
  - `schema` —— 数据库整体设计与表结构(**本次只需导入它**)
  - 另有一份测试用初始数据文件
- 操作:复制 schema 顶部的**建库语句** → 打开 **Navicat** 连接 MySQL → 粘贴执行 → 建库完成(出现 `bloghub` 库)。
- 重新连接到该库 → 粘贴全部建表语句(Navicat 里 **Ctrl+R / Cmd+R** 快速执行所有语句)→ 刷新 → 所有表创建完成。

### 5.3 后端代码生成

| 步骤 | 操作 |
| :--- | :--- |
| 1 | 回 IDEA 新建 Spring Boot 工程;右侧新建数据源,连接刚建好的数据库,测试通过后保存 |
| 2 | 回 Qoder 对话框 **➕ → @database** 引用该数据库 |
| 3 | (可选但推荐)先在个人设置里**新建规则**,导入代码规范 Markdown |
| 4 | 让 Qoder 按库表结构生成后端代码(**Controller / Service / Entity / Mapper** 等) |

**自动纠错过程(亮点):**

1. 生成后自动开始**验证编译** → 报编译错误。
2. 分析出是 **Lombok 与 Java 版本兼容性**问题 → 自动把 Lombok 更新到最新版。
3. 再编译仍有错 → 继续分析 → 修改 **pom 文件** → 再执行 → **编译通过**。
4. 继续修复了一些警告信息,最后给出总结(项目结构、主要 API 接口、基础特性、运行说明)。

**结论:** 几乎全程未手动介入,Qoder **从无到有**生成了一个可成功编译运行的完整后端工程;中途遇错能自动分析并修复,代码质量不错。
