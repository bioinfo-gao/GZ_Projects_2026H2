# Claude Code 七大原语：Harness · Skill · Hook · MCP · Plugin · subAgent · Routine
### 用我们自己的亲历工作做实例的对比教学

> 本文目的：把 Claude Code 里最容易混淆的七个概念**一次讲清区别**，每个都配**我们真实项目里用过的例子**（生信场景）。
> 一句话记忆法在最后。子目录（`Hooks/` `Gmail_MCP/` `plugin/` `Routines/` `skills/` `subgent/` `Harness/`）放各自的细节，本文是顶层索引 + 对比。

## 0. 一张总表（先看这个）

| 原语 | 它本质是什么 | **谁来执行** | **怎么触发** | 作用域 | 我们的实例 |
| :--- | :--- | :--- | :--- | :---: | :--- |
| **Harness（宿主/运行时）** | Claude Code 本体：托管模型、跑工具、管上下文、跑后台任务 | 运行时自己 | 一直在（是舞台本身） | 全局 | tmux 编排、后台哨兵、上下文摘要、权限系统、`/loop`·`/schedule` 的载体 |
| **Skill（技能）** | 一包**指令/工作流**（`SKILL.md`），按需加载改变模型**怎么做** | **模型**读了照做 | 模型自己判断 / 用户打 `/name` | 单次任务 | `/git` `/taxnom` `/tax-assembly-mag` `/tax-functional-humann` `/wgs` `/corun` `superpowers:writing-skills` |
| **Hook（钩子）** | 事件触发时**系统自动跑的代码**（模型跳不过） | **系统**（Claude 或 git） | 某事件发生（提交前、工具调用前…） | 事件级 | 我刚建的 **pre-commit 大文件拦截**（git hook）；settings.json 的 PreToolUse/Stop hook |
| **MCP（Model Context Protocol）** | 把**外部工具/数据源**标准化接进来的协议 | **外部 MCP server** | 模型发起 tool 调用 | 外部集成 | **Gmail MCP**（读/搜/发邮件）、**GitHub MCP**（issue/PR/repo/代码搜索，2026-07-19 加） |
| **Plugin（插件）** | 把 skill+hook+MCP+命令**打包分发**的容器 | —（是分发单元） | 安装后其内容各自生效 | 打包层 | **superpowers 插件**（内含 brainstorming/writing-skills/TDD 等 skill） |
| **subAgent（子代理）** | 主代理派生的**另一个模型实例**，独立上下文干子任务后回传结果 | **另一个模型实例** | 主代理用 Task/Agent 工具派生 | 隔离上下文 | `Explore`（只读搜索）、`general-purpose`、`Plan`；适合并行/隔离重探索 |
| **Routine（定时云代理）** | 按 **cron 定时**自主跑的云端代理（无人在场） | 云端调度器 + 模型 | 到点（时钟） | 定时自治 | `/schedule` 建的每日巡检；区别于 `/loop`（会话内循环） |

## 1. 三条区分主轴（抓住这三条就不混）

1. **谁执行？** 模型(Skill) / 系统(Hook、Harness) / 外部服务(MCP) / 另一个模型(subAgent) / 调度器(Routine)。
2. **是什么？** 指令(Skill) / 事件触发的代码(Hook) / 外部集成(MCP) / 分发包(Plugin) / 工人实例(subAgent) / 定时任务(Routine) / 运行时本身(Harness)。
3. **怎么触发？** 模型判断或用户打命令(Skill) / 事件(Hook) / 模型调用(MCP) / 模型派生(subAgent) / 时钟(Routine) / 常驻(Harness)。

**最关键的一刀**：Skill 是**给模型看的指令**（模型可能不完美地遵守）；Hook 是**系统强制跑的代码**（模型无法绕过）。所以"必须每次都做、不能靠模型自觉"的事→用 **Hook**；"教模型某个流程怎么走"→用 **Skill**。

## 2. 逐个讲 + 我们的实例

### 2.1 Harness（宿主/运行时）—— 舞台
Claude Code 本体：加载模型、执行 Bash/Read/Edit 等工具、管理上下文窗口与摘要、跑后台任务、权限门控。**其余六个都跑在它里面。**
- **实例**：本项目里所有"长任务进 tmux + Phase-1 早失败监控 + 后台哨兵唤醒 agent"都是在利用 harness 的能力；上下文变长时的自动摘要、`run_in_background` 后台命令、工具权限分类器拦截 `git remote set-url`——都是 harness 行为。
- 记忆钩子：harness = 舞台；skill/hook/mcp = 台上的演员和道具。

### 2.2 Skill（技能）—— 教模型"怎么做"
一包 Markdown 指令（`~/.claude/skills/<name>/SKILL.md`），按需加载进上下文，改变模型处理某类任务的方式。分**模型自动可调**和**手动 `/name`**（我们的生信 skill 多是 `disable-model-invocation: true` 手动触发）。
- **实例**：`/tax-assembly-mag`（MAG 组装流程）、`/tax-functional-humann`（HUMAnN 功能谱）、`/git`（git 操作规范 + 大文件排除）、`/corun`（多管线并跑）。还有官方 `superpowers:writing-skills`（教我怎么写 skill）。
- **本质**：Skill 里没有"会自动运行的代码"，它是**提示词层面**的知识与流程；执行者是模型。所以 skill 会"忘"或"变通"——这正是需要 Hook 来兜底强约束的原因。

### 2.3 Hook（钩子）—— 系统在事件点强制跑的代码
**两种，务必分清：**
- **(a) Claude Code hook**（`settings.json` 的 hooks）：harness 在**工具生命周期事件**（PreToolUse/PostToolUse/Stop/UserPromptSubmit…）自动跑的 shell 命令。用于"每次 X 都自动做 Y"——模型跳不过。
- **(b) git hook**（`.git/hooks/` 或 `core.hooksPath`）：**git** 在 git 事件（pre-commit/pre-push）跑的脚本，与 Claude 无关，谁 `git commit` 都会触发。
- **实例**：我刚为 `projects_2026H2` 建的 **pre-commit 大文件拦截**（`.githooks/pre-commit`，暂存文件 >10MB 直接拒绝 commit）就是 **git hook**——这正是"不能靠人/模型每次手动排查大文件"的痛点，交给系统自动检测。
- **Skill vs Hook 一句话**：`/git` skill 是"教我提交前该扫大文件"（靠自觉）；pre-commit hook 是"就算忘了，系统也拦住"（强制）。两者互补。

### 2.4 MCP（Model Context Protocol）—— 标准化接外部世界
一套协议，让外部 **MCP server** 把工具/数据源暴露给模型调用（邮件、数据库、云 API…）。模型像调本地工具一样调 MCP 工具。
- **实例① Gmail MCP**（`mcp__gmail__read_email`/`search_emails`/`send_email`，stdio 传输）——我们用它读客户邮件；但它**下载不了附件**，于是改用它底层的 IMAP 凭据自己写 `imaplib` 脚本抓附件（见 `Gmail_MCP/` 与记忆 `feedback_gmail_attachment_via_imap`）。
- **实例② GitHub MCP**（2026-07-19 加，对生信程序员很重要：自然语言操作 issue/PR/repo/代码搜索，补 `gh` CLI）。两种装法：
  - **(推荐) 远程 hosted + OAuth，不存 PAT**：`claude mcp add --transport http --scope user github https://api.githubcopilot.com/mcp/` → 然后在 Claude Code 里 **`/mcp` 完成 OAuth 授权**（首次会显示 `✘ Failed to connect`，授权后变 `✔`）。需 GitHub 账号（hosted 端）。
  - **(备选) 本地 + PAT**：`claude mcp add --scope user github -e GITHUB_PERSONAL_ACCESS_TOKEN=<你的PAT> -- docker run -i --rm -e GITHUB_PERSONAL_ACCESS_TOKEN ghcr.io/github/github-mcp-server`（本机有 docker；PAT 存进 `~/.claude.json`，该文件不在 git 库、不会被提交）。用于 hosted 不可用或要细粒度 scope 时。
  - 本机 `gh` 走 SSH key、无现成 token，故默认选了远程 OAuth 版；认证那步须你本人做。
- **MCP vs Skill**：Skill 是"内部流程指令"，MCP 是"外部能力接入"。MCP 扩展的是模型**能碰到什么外部系统**，Skill 扩展的是模型**怎么组织工作**。

### 2.5 Plugin（插件）—— 打包分发的容器
把若干 skill + hook + MCP server + slash 命令**捆成一个可安装单元**。插件本身不"执行"，装上后其内含物各自按规则生效。
- **实例**：**superpowers 插件**（`~/.claude/plugins/cache/claude-plugins-official/superpowers/`）——里面装着 `brainstorming`、`writing-skills`、`test-driven-development`、`systematic-debugging` 等一堆 skill。我这次写新 skill 就调用了它的 `writing-skills`。
- **Plugin vs Skill**：Plugin 是**箱子**，Skill 是箱子里的**东西**。一个插件可含多个 skill/hook/MCP。

### 2.6 subAgent（子代理）—— 派生的另一个模型实例
主代理用 Task/Agent 工具派生一个**独立上下文**的模型实例去干子任务，跑完回传结果（主代理上下文不被子任务的海量中间输出污染）。类型有 `Explore`（只读大范围搜索）、`general-purpose`、`Plan` 等。
- **用途**：并行独立子任务、隔离"读一堆文件才得一个结论"的重探索、保护主上下文。
- **实例**：本系列会话我多是**内联**自己干（任务耦合紧、需连续判断），没派子代理；典型该用的场景是"在大 repo 里 fan-out 搜索命名约定"——交给 `Explore` 子代理只回结论。
- **subAgent vs Skill/Hook**：子代理是**另一个工人**（模型实例）；skill 是指令、hook 是代码。子代理会"重新冷启动"、不继承主代理的即时上下文（除非显式传）。

### 2.7 Routine（定时云代理）—— 到点自主跑
按 **cron 定时**在云端自主运行的代理，无人在场也跑。用 `/schedule`（底层 CronCreate）建。
- **实例**：可建"每天早上巡检某流水线/PR"这类；与 `/loop`（**会话内**按间隔重复，如"每 5 分钟查一次部署"）不同——Routine 是**跨会话、时钟驱动、云端**的。
- **Routine vs subAgent**：subAgent 是**当前会话内**派生的即时工人；Routine 是**脱离当前会话、定时**触发的自治运行。
- 注意：本会话里我用的"后台哨兵"(`run_in_background` 的 bash + 到条件唤醒 agent) 是 **harness 后台任务**，不是 Routine——它绑在当前会话生命周期，不是云端 cron。

## 3. 最容易混的四组（考点）

| 容易混 | 区别一句话 |
| :--- | :--- |
| **Skill vs Hook** | Skill=教模型怎么做(靠自觉)；Hook=系统在事件点强制跑的代码(跳不过)。要"必做不可漏"→Hook。 |
| **Skill vs MCP** | Skill=内部流程指令；MCP=接外部系统的标准协议。 |
| **Plugin vs Skill** | Plugin=分发箱子；Skill=箱子里的一件。插件含多个 skill/hook/MCP。 |
| **subAgent vs Routine** | subAgent=会话内派生的即时工人(独立上下文)；Routine=脱离会话、定时(cron)、云端自治。 |
| **Claude hook vs git hook** | 前者 harness 在工具事件跑(settings.json)；后者 git 在 git 事件跑(.githooks)。都叫 hook，触发源不同。 |

## 4. 一句话记忆法

- **Harness**＝舞台（一切在其上）
- **Skill**＝剧本（教演员怎么演，演员=模型，可能演走样）
- **Hook**＝自动机关（到点必触发，演员管不着）
- **MCP**＝对外的门（通往邮件/数据库/API）
- **Plugin**＝道具箱（一箱装剧本+机关+门）
- **subAgent**＝替身演员（另派一个，独立记忆）
- **Routine**＝闹钟（到点自己上场，哪怕没人看）
