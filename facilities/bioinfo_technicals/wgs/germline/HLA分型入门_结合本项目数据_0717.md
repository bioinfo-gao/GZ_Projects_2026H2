## 1. HLA 是什么，为什么要分型

**HLA（Human Leukocyte Antigen）= 人类的 MHC（Major Histocompatibility Complex）**，位于 **chr6p21**（约 chr6:28–34 Mb 的扩展 xMHC 区）。它编码的蛋白负责**把细胞内/外的多肽"举给"T 细胞看**，是免疫系统识别"自我 vs 非我"的核心。

它是**人类基因组里最多态的区域**：单一 HLA-B 基因在人群里就有 **上万个** 已知 allele。这种极端多态正是分型难的根源，也是它有用的原因：

| 应用场景 | 为什么需要 HLA 分型 |
| :--- | :---: |
| 器官/造血干细胞移植 | 供受体 HLA 越匹配，排斥/GvHD 越轻——移植配型的核心 |
| 疾病关联 | 很多自身免疫病与特定 allele 强关联（如 B\*27 与强直性脊柱炎、DQ2/DQ8 与乳糜泻） |
| 药物超敏（pharmacogenomics） | B\*57:01→阿巴卡韦超敏、B\*15:02→卡马西平 SJS，用药前需查 |
| 免疫/肿瘤 | 新抗原（neoantigen）预测、疫苗设计都要先知道个体 HLA type |
| 亲缘/身份 | HLA 单倍型按块遗传，可辅助判断亲缘关系 |

> **本项目**：客户没说组织来源，HLA 分型是"if feasible"的附加交付项；这里主要作为样本刻画的一部分，不是为移植配型做的临床级分型。

## 2. HLA 基因家族地图（用本项目 41 个 T1K 位点当实例）

T1K 一次报 41 个位点，分几类。本项目两样本各有 **37/41** 位点返回了至少一个 allele：

| 类别 | 基因 | 功能 | 递呈给 | 本项目情况 |
| :--- | :--- | :--- | :---: | :--- |
| **经典 class I** | HLA-**A**, **B**, **C** | 递呈**内源**多肽（病毒、肿瘤） | CD8⁺ T | **最常用，但本数据里 A/B 测不准（见 §5）** |
| **经典 class II** | HLA-**DRB1**, **DQA1/DQB1**, **DPA1/DPB1** | 递呈**外源**多肽 | CD4⁺ T | **本数据里最可靠的部分** |
| class II 辅助 | DRB3/DRB4/DRB5 | 随 DRB1 单倍型连锁出现 | CD4⁺ T | 本数据 DRB3 高置信、DRB4 只能测"存在"、DRB5 缺如 |
| 非经典 class I | HLA-E, F, G | 免疫调节（NK 等） | — | 本数据都测到，但临床意义有限 |
| 假基因/低表达 | HLA-H, J, K, L, N, P… | 多不编码功能蛋白 | — | 报告不关注 |

**要点**：临床/研究最关心的是 **A、B、C（class I）+ DRB1、DQB1、DPB1（class II）** 这 6 个经典位点。本项目里 **class II 那几个可靠、class I 的 A/B 不可靠**——恰好和直觉相反，原因见 §5。

## 3. HLA 命名法（nomenclature）—— 拆解本项目的真实 allele

HLA allele 名字形如 **`HLA-A*23:144`**，用冒号分**字段（field）**，每段含义不同。以本项目真实等位为例：

```
HLA-DRB1 * 13 : 03 : 01 : 02
        │   │    │    │    │
        │   │    │    │    └─ field4: 非编码区（内含子）差异
        │   │    │    └────── field3: 同义突变（synonymous，蛋白不变）
        │   │    └─────────── field2: 蛋白序列差异（决定不同的抗原）★最关键
        │   └──────────────── field1: allele group（血清型大类，如 DR13）
        └──────────────────── 基因名
```

- **field1（大类）**：如 `A*23`、`DRB1*13` —— 相当于老式血清学分型的"大组"。
- **field2（2-field，如 `A*23:144`）**：**蛋白水平**分型 —— 这是绝大多数应用真正需要的分辨率。两个 allele 只要 field1:field2 相同，蛋白就一样。
- **field3/field4**：只是同义/内含子差异，**功能上通常无所谓**。所以报告里看到 `DPA1*02:01:01`，真正重要的是 `02:01` 那两段。
- **后缀字母**：本项目出现过 `DRB4*01:199N` 的 **`N`** = **Null allele（不表达蛋白）**；其他还有 `L`(低表达)、`S`(可溶) 等。看到 `N` 要知道"这个 allele 即便在，也不产功能蛋白"。

> **分辨率术语**：只给到 field2 叫 **2-field / two-digit-pair resolution**（蛋白级）；给到 field3/4 叫 high-resolution。**WGS 通常只能可靠到 2-field，甚至 class I 连 2-field 都勉强**（见 §5）。

## 4. WGS 怎么做 HLA 分型，为什么难

**流程（本项目 T1K 的做法，见 `scripts/6_hla_typing.sh`）**：

1. 从全基因组 CRAM 里**抠出 MHC 区**（chr6:28–34 Mb）的 reads + **unmapped reads**（有些 HLA reads 因为和参考单倍型差异太大而没比对上）。
2. 转成 FASTQ，喂给专门的 HLA genotyper（本项目用 **T1K**，`--preset hla-wgs`）。
3. genotyper 把 reads 比到 **IPD-IMGT/HLA 数据库**（所有已知 allele 序列），用 EM 之类的算法估计"最可能是哪两个 allele"。

**为什么难（这也是为什么要看 quality）**：

| 难点 | 后果 |
| :--- | :---: |
| **极端多态**：exon 2/3（class I）/exon 2（class II）编码抗原结合槽，人群差异巨大 | 短 reads 难唯一确定是哪个 allele |
| **高度同源**：A/B/C 彼此、以及和一堆假基因序列相似 | reads 容易"串位"到错的基因 |
| **参考偏倚（reference bias）**：GRCh38 只含一条 HLA 单倍型 | 和参考差异大的 allele 的 reads 更容易丢/误比 → 常偏向常见 allele、或第二 allele dropout |
| **WGS 深度有限**：全基因组 ~25×，摊到 MHC 高多态区的**有效**深度更低 | class I（尤其 A/B）覆盖不足、等位难分 |

> 专门的**扩增子（amplicon）/长读长 HLA 分型**（PacBio/ONT + SBT）能到高分辨率，但那是靶向实验，不是从 WGS 顺带能得到的。**WGS HLA 分型 = 筛查级，不是金标准**。

## 5. 读懂 T1K 输出 —— 用本项目数据逐列讲

`hla_typing/<sample>_hla_genotype.tsv` 每行一个基因，列的含义：

```
gene   num_alleles   allele1   abundance1   quality1   allele2   abundance2   quality2   [更多候选]
```

- **num_alleles**：这个基因测到几个 allele（0/1/2）。
- **allele**：allele 名（可能是**逗号分隔的一串候选**，见下）。
- **abundance**：该 allele 的归一化读段支持/丰度。可靠 call 通常 ~30–50，弱 call 只有 ~1–4。
- **quality**：**T1K 的置信分，本数据范围 0–60，越高越可信**；`-1` 表示没有该 allele。**这是读结果最关键的一列**。

### 用本项目 Sample_A 的真实行举例

| gene | allele1 (q) | allele2 (q) | 怎么读 |
| :--- | :--- | :--- | :--- |
| HLA-**C** | C\*07:18:01 **(q21)** | .（无） | 高置信，但**只一个 allele** → 纯合 or dropout（分不清） |
| HLA-**A** | A\*23:144 **(q1)** | A\*01:01:01 **(q0)** | **两个都 q≈0 → 不可信**，具体数字别当真 |
| HLA-**B** | B\*15:16:01 **(q0)** | B\*35:03:01 **(q0)** | 同上，**低置信** |
| HLA-**DPA1** | DPA1\*02:01:01 **(q36)** | DPA1\*01:03:01 **(q34)** | 两个都高 q → **可靠** |
| HLA-**DRB4** | DRB4\*01:01:01,**…52 个候选…**(q0) | . | "DRB4 **存在**，但**具体 allele 分不清**" |

**三个必须会读的信号**：

1. **quality 高低分层**：q≥13 可用；q=0–1 基本是"猜的"，**别把低 q 的 allele 数字写进结论**。
2. **单 allele（num_alleles=1）**：可能真纯合，也可能第二个 dropout —— **不能直接说"纯合"**。
3. **逗号候选串**：T1K 把无法区分的 allele 全列出，**第一个是代表**。这不是错误，是如实表达"分不清"。

## 6. 结合本项目数据的完整解读

### 6.1 分层可靠性（这批数据的核心结论）

| 层级 | 位点 | quality | 能不能用 |
| :--- | :--- | :---: | :---: |
| **可靠** | class II：DRB1, DQB1, DPA1, DPB1, DRB3；HLA-C；Sample_B 的 HLA-A | 13–36 | ✅ 可用 |
| **低置信** | class I：HLA-A(Sample_A)、HLA-B(两样本) | 0–1 | ⚠ 需正交确认 |

**反直觉点**：一般印象里 class I（A/B/C）是"最经典最该准的"，但**本数据恰恰是 A/B 最不准**。原因就是 §4 说的——class I 抗原结合槽超多态、WGS 有效深度不够。这提醒你：**HLA 结果的可信度是"逐位点"的，不能一刀切说"HLA 都测出来了"**。

### 6.2 单倍型自洽性 —— 一个"结果是真的"的旁证

两样本都出现 **DRB1\*13 + DRB3\*02:02:01 + 无 DRB5（num_alleles=0）**。这不是巧合：**DRB1\*13 单倍型在遗传上就连锁 DRB3（DR52 群），而不连 DRB5**。DRB3/4/5 这三个辅助基因的"有没有"是随 DRB1 大类固定的：

- DR3/11/12/**13**/14 → 带 **DRB3**
- DR4/7/9 → 带 DRB4
- DR15/16 → 带 DRB5

本数据 DRB1\*13 → 有 DRB3、没 DRB5 —— **完全符合遗传规律**。这种内部一致性是"class II 结果可信"的有力旁证（假的、噪声拼出来的结果不会自洽）。

### 6.3 跨样本共享（本项目已写进报告 §6.5 的中性观察）

两样本**共享**若干高置信 allele，甚至 **DPA1 基因型完全相同**（\*02:01:01 + \*01:03:01）；也共享 C\*07:18:01、DRB3\*02:02:01、DPB1\*04:01:01、DQB1\*03:03:02。

但它们在**另一些高置信位点又明确不同**（HLA-A、DRB1 首等位、DQB1 另一等位）——**所以不是同一个人**（同一个体会在所有高置信位点全等）。这种"部分共享"符合两种可能：**亲缘个体共享一条 HLA 单倍型**，或**只是碰巧共享常见 allele**。报告里已中性提示：若客户预期两样本无关，建议核对样本来源。

> 这也顺带示范了 HLA 的一个用途：**HLA 按单倍型块遗传，共享模式可提示亲缘**——但要下结论必须用高置信位点，且最好靠靶向高分辨分型。

## 7. 什么时候、怎么做确认

WGS HLA 分型是**筛查级**。当结果要用于**有后果的决定**（移植配型、B\*57:01 用药、疾病关联研究入排）时，低置信位点必须正交确认：

| 方法 | 分辨率 | 场景 |
| :--- | :---: | :---: |
| **amplicon-based SBT**（Sanger/NGS 靶向测序） | 高（3–4 field） | 临床金标准，逐位点 |
| **长读长 HLA typing**（PacBio/ONT 全长基因） | 最高，可定相（phasing） | 全 allele + 单倍型 |
| 专用 WGS 工具交叉验证（HLA-LA、OptiType、Arcas 等） | 与 T1K 相当 | 多工具一致性提高可信度，但都受制于 WGS 深度 |

**本项目的具体建议**（已写进报告）：class II + HLA-C 可放心用；**HLA-A/B 若要用，先做靶向确认**。

## 8. 一页速查 + 常见误区

**读 T1K HLA 结果的正确姿势**：
1. 先看 **quality**，再看 allele 名 —— q=0–1 的 allele 数字**不作数**。
2. **单 allele ≠ 纯合**（可能 dropout）。
3. **逗号串 = 分不清**，取第一个当代表，别当成"测到很多"。
4. 关心 **2-field**（蛋白级）就够；field3/4 差异通常无所谓；后缀 **N = 不表达**。
5. class I（A/B）在 WGS 常不准，**别默认"经典位点最可靠"**。
6. 用**单倍型连锁规律**（DRB1↔DRB3/4/5）给自己做一次自洽性检验。

**常见误区**：
- ❌ "HLA 都测出来了" → 应说"逐位点看，某些可靠、某些低置信"。
- ❌ 把 `A*23:144` 的 `:144` 当确定结论 → 那是 q=1 的猜测。
- ❌ 看到 num_alleles=1 就写"纯合" → 可能是 dropout。
- ❌ 拿 WGS HLA 结果直接做移植配型 → 必须靶向金标准确认。
