`.gitignore` 里 `*/output_results/` 和 `**/output_results/` 看起来只差一个星号，实际匹配
范围天差地别——前者只能忽略"仓库根目录往下数第二层"的 `output_results/`，后者能忽略
**任意深度**的。这份笔记记录原因，配实测验证（不是背文档，是真的搭了个沙盒仓库跑
`git check-ignore` 测出来的）。

## 背景：真实踩过的坑

项目仓库里原来写的是：
```
*/output_results/
```
本意是"忽略所有 sarek/nextflow 项目的 `output_results/` 输出目录"。这条规则对
`13_Ellen_knockin_wgs/output_results/` 生效，但对 `13_Ellen_knockin_wgs/BAC/output_results/`
（多嵌套了一层 `BAC/`）完全没生效——导致一份 9.6GB 的 pipeline 输出被 git 意外跟踪提交。
改成 `**/output_results/` 之后，任意深度都能覆盖。

## 实测验证（搭沙盒仓库跑出来的真实结果）

三层测试目录：`proj/output_results/`（1层）、`proj/BAC/output_results/`（2层）、
`proj/BAC/deep/output_results/`（3层）。

| gitignore 规则 | 1层 (`proj/output_results/`) | 2层 (`proj/BAC/output_results/`) | 3层 (`proj/BAC/deep/output_results/`) |
| :--- | :---: | :---: | :---: |
| `*/output_results/` | ✅ 忽略 | ❌ **不忽略** | ❌ **不忽略** |
| `**/output_results/` | ✅ 忽略 | ✅ 忽略 | ✅ 忽略 |
| `output_results/`（对照，不带任何斜杠） | ✅ 忽略 | ✅ 忽略 | ✅ 忽略 |

## 原理：两条独立的规则叠加在一起，缺一个都会错

gitignore 的匹配行为由**两条互相独立的规则**共同决定，`*` 和 `**` 的差异只是其中一条：

### 规则A：`*` 只匹配"一层路径内的任意字符"，不会跨越 `/`

`*` 是"通配符"没错，但它的通配范围**严格限制在一个路径段以内**，遇到 `/` 就截止。
所以 `*/output_results/` 里的 `*`，只能代表**恰好一层目录名**，不能代表"一层或多层"。

### 规则B：只要 pattern 里出现了"非末尾位置"的 `/`，这条 pattern 就会被"锚定"在
`.gitignore` 所在的目录上，不会自动在每一级子目录里重新生效

这是最容易被忽略的一条，也是这次踩坑的真正原因。git 官方文档（`man gitignore`）的原话
（大意）：如果 pattern 里在**开头或中间**出现了 `/`，这条 pattern 就只相对于该 `.gitignore`
文件所在的目录生效（即"锚定"，只在这一个起点上匹配，不会自动往下钻）；如果 pattern **完全
不含 `/`**（末尾的 `/` 不算，那只是"仅匹配目录"的标记，不算"中间的/"），则该 pattern 会被
当成"在任意目录层级都可能出现"来处理，git 内部等价于自动在它前面加了一个 `**/`。

把这两条规则套到我们的三个 pattern 上：

- **`*/output_results/`**：中间有一个 `/`（在 `*` 和 `output_results` 之间）→ 触发规则B，
  被"锚定"在仓库根目录，只从根目录这一个起点匹配；而规则A又决定 `*` 只能吃掉**一层**，
  于是这条 pattern 实际的意思是"仓库根目录正下方一层、名叫任意名字的目录，里面的
  `output_results/`"——**深度写死为2层**（根 + 1层 + output_results），既不会浅一层，
  也不会深一层。
- **`**/output_results/`**：`**` 被 git 定义为特殊记号，专门表示"零层或多层目录，允许跨越
  `/`"——所以它不受规则A的"单层"限制，可以吃掉任意多层路径；同时它作为**开头**的 `**/`，
  文档里明确写了"匹配所有目录"，等价于对每一级子目录都重新套用一次 `output_results/`
  这条规则。
- **`output_results/`**（无任何斜杠，只有末尾那个不算数）：因为完全不含"中间的 `/`"，
  直接触发规则B里"未锚定"的分支，效果和 `**/output_results/` 一样，任意深度都匹配。
  ——这是个更简洁的写法，如果确实想要"叫这个名字的目录，不管出现在哪一层都忽略"，
  直接写 `output_results/` 就够了，不必特意加 `**/`。`**/output_results/` 的价值在于
  **更明确地表达"我是故意要任意深度都匹配"的意图**，读代码的人不用去查 gitignore 的
  锚定细则就能看懂。

## 一句话记住

`*` 管的是"这一层通配"，`/` 的**位置**管的是"要不要锚定在根目录、还是允许在任意层级重新
套用"。两者独立叠加，`*/xxx/` 因为中间带了 `/` 被锚定、又只能吃一层，深度被写死；
`**/xxx/` 或裸的 `xxx/` 都不带这种"中间 `/` 锚定"的副作用，能在任意深度生效。
