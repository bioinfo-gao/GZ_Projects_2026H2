## 二、`5B_annotation.R`（GTF转基因注释表）

这个文件不是小修小改，是**整体重写**，因为原脚本本身就有两个问题，不改的话根本跑不起来：

1. **路径是占位符**：`GTF_FILE <- "T_majus.final.gtf"` —— 这文件不存在（这是植物项目的占位符文件名，从没被换成真实路径过）。
2. **逻辑被注释掉、且有变量未定义的bug**：核心的基因表格构建代码（`gene_df <- ... filter(type=="gene")` 那一段）全部被注释掉了，但脚本最后一行 `cat("📊 共提取", nrow(client_df), ...)` 仍然引用 `client_df` 这个变量——这个变量从未被赋值，**只要按原样运行这个脚本必定直接报错退出**。

修改内容：

| 编号       | 改了什么                                           | 原值/原问题                                       | 新值/解决方法                                                                                                                | 为什么                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| ---------- | -------------------------------------------------- | ------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| MODIFIED-1 | 新增 `setwd()`                                   | 没有设置工作目录                                  | 设为 `script_opposum/`                                                                                                     | 保证相对路径输出位置正确                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| MODIFIED-2 | `GTF_FILE`                                       | 占位符 `"T_majus.final.gtf"`（不存在）          | `/Work_bio/references/Didelphis_virginiana/mDidVir1/DNA_Zoo/Didelphis_v.liftoff.gtf`                                       | 这是你跑 nf-core/rnaseq 流程时实际用的GTF（我从 `output_results/pipeline_info/params_*.json` 里的 `gtf` 参数核对过，和流程用的完全一致），保证注释表和你的表达矩阵用的是同一套基因坐标体系                                                                                                                                                                                                                                                                                          |
| MODIFIED-3 | `OUT_DIR`                                        | 当前目录                                          | `../Data_Analysis`                                                                                                         | 跟 `4_run_DE_PCA_final.R` 的输出目录统一，方便你一个文件夹里找所有结果                                                                                                                                                                                                                                                                                                                                                                                                                |
| MODIFIED-4 | 输出文件名                                         | `T_majus_Gene_Annotation_Client.csv`            | `Didelphis_virginiana_Gene_Annotation_Client.csv`                                                                          | 按实际物种命名                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| MODIFIED-5 | **核心聚合逻辑重写**（最关键的改动）         | 按 `type=="gene"` 过滤，整段被注释掉            | 改为按 `transcript` 行聚合到基因层级                                                                                       | 我读取这个GTF文件实际检查后发现：它**根本没有 `"gene"` 这个feature类型**，只有 `transcript`/`exon`/`CDS`（这是liftoff这类同源迁移注释工具的常见特点，它不输出gene级别的合并行）。所以原来"按type=='gene'过滤"的写法，就算把注释去掉也只会得到0行结果，必须改成自己从transcript行按基因ID聚合出坐标范围                                                                                                                                                                    |
|            | 同一改动里，**额外发现并修复了一个数据陷阱** | （原脚本没有处理这个问题，因为它压根没跑起来过）  | 按 `(gene_id, 染色体/scaffold)` 先聚合，每个gene_id挑转录本数最多的那个locus作为代表，并加一列 `n_loci` 标记有歧义的基因 | 实测发现这份liftoff注释里有**415个基因（约1.5%）的gene_id同时出现在两个不同的scaffold上**（例如 `KCTD1` 同时在 `HiC_scaffold_1` 和一个长度仅989bp的小scaffold `HiC_scaffold_22483` 上）。如果直接按gene_id取全局最小/最大坐标，会把两个完全不同scaffold的位置硬拼成一个跨度上亿bp的"假基因"，坐标完全没有生物学意义。现在的处理方式是：保留"转录本数最多"的那个位置作为该基因的主要注释，同时用 `n_loci`列诚实地标记出这415个有歧义的基因，方便你后续如果需要可以手动核查 |
| MODIFIED-6 | 末尾增加统计提示                                   | 原代码引用了未定义的 `client_df`相关变量（bug） | 改为正确统计并打印 `n_loci>1` 的基因数                                                                                     | 修复运行时必崩溃的问题，并把"多locus基因有多少个"这个信息明确告诉使用者                                                                                                                                                                                                                                                                                                                                                                                                                 |

**已验证**：脚本已实际跑通，输出 27,668 个基因的注释表到

其中 415 个基因被标记为 `n_loci > 1`（多候选位点）。

## 三、需要你确认/判断的地方（不是我能单方面决定的）

1. **`5B_annotation.R` 里 415个多locus基因的处理方式**：我目前的策略是"自动选转录本数最多的locus，丢弃另一个"。如果这些基因恰好是你研究里关心的基因，建议你打开CSV筛选 `n_loci > 1` 的行，人工核实一下选的locus对不对（尤其检查是不是真的基因家族重复，而不是liftoff转移错误）。
2. **旧注释遗留问题**：`4_run_DE_PCA_final.R` 里那段"本实验出现一个意外结果..."的注释是否要删，我没有处理，留给你决定。
3. **`4_run_DE_PCA_final.R` 目前只做了 `pi5 vs NC` 一个对比**——如果你后续还有其他分组比较需求，需要在 `contrasts` 列表里追加。
