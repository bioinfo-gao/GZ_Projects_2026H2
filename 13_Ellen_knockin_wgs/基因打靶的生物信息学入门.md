基因打靶构建体 WT+KI 序列(.dna) 什么意思

=====

这句话是一个典型的分子生物学和基因工程领域的专业术语组合。简单来说，它指的是一个用于“基因敲入（Knock-in）”实验的DNA载体（质粒）的序列文件。


为了让你完全理解，我们可以将这句话拆解为三个部分来详细解释：

1. 基因打靶构建体 (Gene Targeting Vector / Construct)
   基因打靶：是一种通过同源重组（Homologous Recombination）技术，对生物体基因组中的特定基因进行精确修饰的技术。它可以实现基因敲除（Knock-out）、敲入（Knock-in）或点突变等。
   构建体（载体）：指研究人员在体外人工构建的DNA分子（通常是质粒）。这个质粒里包含了实现打靶所需的所有“零件”，通常包括：
   同源臂（Homology arms）：与目标基因组序列完全一致的DNA片段，用于引导质粒精准定位到基因组的特定位置。
   筛选标记（Selection marker）：如抗性基因（新霉素抗性、嘌呤霉素抗性等）或荧光蛋白，用于筛选成功整合了质粒的细胞。
   打靶核心序列：需要插入或替换到基因组中的目标序列。
2. WT+KI (Wild-Type + Knock-In)
   这是该构建体的具体设计策略或命名：
   WT (Wild-Type，野生型)：通常指代构建体中包含的野生型基因组同源臂序列，或者指代该构建体旨在保留目标基因的野生型表达框架/启动子。
   KI (Knock-In，敲入)：指将外源基因、报告基因（如GFP/Luciferase）、蛋白标签（如Flag/HA）或特定的突变序列，精准插入到基因组的特定位点。与“敲除（破坏基因）”不同，“敲入”是在保留或修饰原有基因的基础上“添加”东西。
   WT+KI 组合含义：在商业公司（如小鼠模型构建公司）或实验室的内部命名中，这通常表示该构建体是用于在野生型基因位点进行敲入，或者该质粒序列中同时包含了“野生型同源臂（WT）”和“需要敲入的目的片段（KI）”。（有时也指打靶后预期获得野生型和敲入型等位基因共存的杂合状态）。
3. 序列(.dna)
   这指的是上述构建体的具体DNA序列数据文件。
   .dna 格式：这是一种特定的生物信息学软件文件格式。最常见的是 SnapGene 软件的专属格式。
   文件内容：与普通的纯文本序列（如 .fasta）不同，.dna 文件不仅包含A/T/C/G碱基序列，还包含了丰富的注释信息，例如：
   质粒的环形图谱（Map）。
   限制性内切酶位点。
   开放阅读框（ORF）、启动子、终止子、同源臂的具体位置。
   引物结合位点等。
   💡 实用建议：如何打开这个文件？
   如果你拿到了这个 .dna 文件，普通的文本编辑器（如记事本）打开会是乱码。你需要使用专业的序列分析软件来查看：
   SnapGene（最推荐）：.dna 是它的原生格式，打开后可以直接看到漂亮的质粒环形图谱和详细的序列注释。
   ApE (A Plasmid Editor)：一款免费开源的软件，支持打开 .dna 格式。
   在线转换/查看：如果你没有软件，可以将文件上传到一些在线生物工具，或者请有 SnapGene 软件的同事帮你将其另存为通用的 .gb (GenBank) 或 .fasta 格式。
   总结
   你看到的这个文件，是研究人员为了在细胞或实验动物（如小鼠）的特定基因位点上 “敲入（KI）”一段外源序列，而精心设计的一个打靶质粒（包含野生型同源臂等元件） 的数字化序列图纸。它是进行后续细胞转染、显微注射和阳性克隆筛选的核心物质基础。

   含义： 筛选标记仍然留在小鼠的基因组里，位于你敲入的 CD1A 基因附近。
   影响： 有时 Neo 盒自带的强启动子会干扰邻近基因（即 CD1A 或周围基因）的正常表达，导致表型不纯或产生副作用。

   含义： 理论上应该切除，但 WGS 发现还有 Neo 序列的 reads，或者切除不完全。这属于基因型质量控制（QC）失败。

在 WGS（全基因组测序）分析中，观察到 Reads 跨过 Neo 盒的边界（Junctions）是一个核心判定证据。根据你的项目背景，这通常意味着以下几种结论：

1. 结论一：确认“位点特异性打靶”成功 (Targeted Integration)
   如果你观察到的 Reads 一端比对在 Neo 盒序列上，另一端准确地比对在小鼠基因组的预期插入位点（同源臂外侧）：

   结论： 打靶位置正确（Correct Locus）。 这是证明 Knock-in (KI) 成功的金标准证据之一。
2. 结论二：确认 Neo 盒为“残留/存在”状态 (Neo-Included)
   如果你观察到大量 Reads 完整地覆盖了 左同源臂-Neo连接处 和 Neo-右同源臂连接处：
   意义： 物理结构上，Neo 盒依然完整地镶嵌在基因组中。
   结论： 该样本为 Neo-Positive 状态。
   如果客户的设计目标就是要保留 Neo，那么这是符合预期的。
   如果客户声称已经通过 Cre 酶切除了 Neo，那么这是实验失败（Neo 未切除或切除不完全）。
3. 结论三：排除“随机整合” (Excluding Random Integration)
   在转基因过程中，构建体有时会随机掉进基因组的其他地方。
   分析： 如果你只在预期的 Junction 位点看到跨界 Reads，而在基因组其他地方找不到 Neo 的边界：
   结论： 无随机整合（No Random Integration）。 这是一个非常积极的质量控指标，说明该小鼠品系非常“干净”。
4. 结论四：判定合子型 (Zygosity)
   观察 Junction 处的 Reads 比例：
   纯合子 (Homozygous)： 如果该位点所有的 Reads 都显示有 Neo 插入，没有发现 WT（野生型）的完整序列 Reads。
   杂合子 (Heterozygous)： 如果你既看到了“跨过 Neo 边界的 Reads”，又看到了“完整的、未断开的野生型 WT Reads”。
   结论： 可以判定该小鼠是纯合体还是杂合体。

   "By identifying reads spanning the Neo cassette junctions, we confirmed the site-specific integration of the CD1A construct and verified that the Neo marker remains present at the target locus."

在生物信息学中，分析 loxP 位点是验证基因编辑（尤其是 Cre-lox 系统）是否成功的核心任务。以下是一份针对生信工程师的 loxP 位点分析实战指南。

1. 核心背景知识：loxP 序列特征
   loxP 是一个长度为 34 bp 的特定序列，结构如下：
   序列： ATAACTTCGTATA - GCATACAT - TATACGAAGTTAT
   组成：
   两个 13 bp 的反向重复序列（Inverted Repeats，即结合位点）。
   中间 8 bp 的间隔区（Spacer/Spacer region，决定方向性）。
   功能： 两个方向相同的 loxP 位点在 Cre 酶作用下，会切除中间的片段，并留下一个 “疤痕”（Scar），即一个残余的 loxP 位点。
2. 生信分析目标
   在 WGS 或扩增子测序中，你的目标通常是：
   确认 loxP 的位置和数量： 是否准确插入到了预定坐标？
   验证切除效率（KO 验证）： 两个 loxP 之间的片段（如 Neo 盒或外显子）是否已被切除？
   检查方向： 方向是否一致（决定了是切除还是翻转）？
3. 分析流程与方法
   第一步：构建自定义参考基因组 (Reference Augmentation)
   标准参考基因组（如 mm10/mm39）不含有你人工设计的 loxP 序列。
   操作： 从你的 SnapGene (.dna) 文件中提取含有 loxP 的 KI 构建体序列。
   技巧： 如果是做 WGS 比对，建议创建一个“人工染色体”或者将 KI 序列加入到参考基因组的 FASTA 文件中，防止 Reads 因为找不到匹配而变成 Unmapped。
   第二步：识别“跨界 Reads” (Junction Analysis)
   这是证明 loxP 存在的关键证据。
   寻找 Split-reads： 一部分比对在小鼠基因组（WT），另一部分比对在 loxP 序列上的 Reads。
   寻找 Discordant pairs： 一端 Read 在 WT 基因组，另一端 Read 在 loxP 内。
   第三步：判定 Cre 酶切除结果
   这是你目前最可能遇到的场景：
   场景 A：切除前 (Floxed/Neo-Included)
   你会看到两个 loxP 位点的 Junction Reads。
   中间片段（如 Neo 盒）有完整的 Coverage（覆盖度）。
   场景 B：切除后 (Deleted/Neo-Removed)
   关键特征： 你会发现中间片段（Neo）的 Coverage 归零。
   决定性证据： 观察到一种新的 Junction Read：它的左侧是左同源臂，右侧直接跳过中间片段，连到了残留的那个 loxP 序列及右同源臂上。
4. 常用工具链
   序列查找： seqkit locate 或 grep
   直接在 Fastq 中查找 loxP 序列：grep "ATAACTTCGTATA" sample.fastq。
   比对分析： BWA-MEM 或 Bowtie2
   注意：loxP 只有 34bp，比对参数要允许较短的 Match。
   可视化： IGV (Integrative Genomics Viewer)
   必做： 将 Reads 比对到你的自定义参考序列上，在 IGV 中手动查看 loxP 插入点的 Soft-clipping（软切断）情况。
   结构变异预测： Manta 或 Delly
   这些工具可以自动识别这种“切除”导致的缺失（Deletion）。
5. 常见坑点（QC 注意事项）
   多重复位点： 如果基因组中有多处 loxP（比如同时有两个基因在做 Flox），Reads 可能会发生错比（Mis-mapping）。
   引物二聚体/污染： loxP 是人工合成序列，有时测序污染也会出现该序列，必须通过 Junction Reads（跨边界比对） 来证明它确实在基因组里。
   嵌合体（Mosaicism）： 在 F0 代小鼠中，可能部分细胞切除了，部分没切除。在生信上表现为：既能看到切除后的 Junction，又能看到中间片段（Neo）的低覆盖度 Reads。

基因打靶（Gene Targeting）中，FRT（Flippase Recognition Target）位点与 loxP 位点的作用非常相似，但它是由 Flp 酶识别的。
在你的周报提到的项目中，FRT 通常被用来切除 Neo 盒（Neo cassette）。以下是针对 FRT 的生物信息学分析指南。

1. FRT 序列特征
   FRT 位点也是一个长度为 34 bp 的对称序列：
   序列： GAAGTTCCTATTC - TCTAGAAA - GTATAGGAACTTC
   结构：
   13 bp 反向重复序列（结合位点）。
   8 bp 核心间隔区（决定方向）。
   应用场景： 客户通常在 Neo 盒的两端放置 FRT。当小鼠与表达 Flp 酶的小鼠交配后，FRT 之间的 Neo 盒会被切除，留下一个 FRT “疤痕”。
2. 生信分析目标
   作为生信工程师，你在处理 WGS 数据时，针对 FRT 的目标是：
   定位 FRT： 确认 FRT 插入的位置是否在设计的内含子或非编码区。
   验证切除 (Deletion)： 证明 Neo 盒是否已被 Flp 酶彻底清除。
   检查残留： 确认是否只留下了一个 FRT 疤痕，且序列无突变。
3. 分析流程
   第一步：准备参考序列 (Reference Preparation)
   在你的 Positron/Python 环境中，从 .dna 文件提取序列：
   如果项目标记为 "Neo-Removed"：你的参考序列应该是：左同源臂 -- FRT疤痕 -- 右同源臂。
   如果项目标记为 "Neo-Included"：参考序列应该是：左同源臂 -- FRT -- Neo盒 -- FRT -- 右同源臂。
   第二步：寻找 FRT 边界 Reads (Junction Reads)
   在 BWA 比对后的 BAM 文件中，利用 IGV 或脚本寻找以下证据：
   Soft-clipping Reads： Read 的一部分比对在基因组，另一部分因为比对不上 FRT 序列而被“切除”。这些 Read 标记了插入的精准点。
   Split-reads： 同一条 Read 跨过了 小鼠序列 和 FRT序列 的交界处。
   第三步：判定切除状态 (Flp-mediated Recombination)
   这是最关键的逻辑判定：
   结论：未切除 (Intact Neo)
   在 IGV 中，你能看到 Read 覆盖了完整的 Neo 基因序列。
   你能找到 FRT - Neo 的连接处序列。
   结论：已切除 (Neo Deleted)
   覆盖度断崖： Neo 盒区域的 Read Coverage 突然降为 0。
   新连接处： 你能找到一条 Read，左边比对在第一个 FRT 之前，右边直接跨过整个 Neo 空间，比对到了残留的 FRT 疤痕和之后的序列。

3. 处理“同源臂”干扰：
由于同源臂序列与小鼠基因组完全一样，Reads 会优先比对到基因组。
技巧： 如果要确认 FRT 是否插入成功，必须寻找那些一头在同源臂内，一头在 FRT 序列内的 Read 对（Discordant Pairs）。

4. loxP 与 FRT 的双重系统 (Dual System)
在复杂模型（如你处理的 CD1A 或 Lats1/2）中，客户常同时使用两者：
FRT 用于删抗性： 用 Flp 酶切掉 Neo 盒，使基因组“干净”。
loxP 用于删基因： 用 Cre 酶在特定组织（如卵巢肿瘤）中切掉关键外显子（Exon）。
你在 WGS 分析中的结论模板：
"WGS analysis confirms the successful excision of the Neo cassette via Flp-FRT recombination. We observed a complete loss of coverage across the Neomycin gene body and identified high-confidence junction reads connecting the flanking genomic regions via a single residual FRT site (scar)."


5. 常见问题 (Troubleshooting)
FRT 突变： 有时在合成或合成过程中，FRT 的 34bp 会发生 1-2 bp 突变，这可能导致 Flp 酶切除效率下降。在 IGV 中需注意查看 FRT 区域是否有明显的变异位点（Mismatch）。
嵌合 (Mosaicism)： 如果是来自 F0 代的样本，你可能会在同一个位置既看到“带 Neo”的 Reads，又看到“不带 Neo”的 Reads。这意味着小鼠身体里只有一部分细胞发生了重组。
